		八种HTTP方法
GET：最常见的，像服务器请求信息，参数会跟在url后面，所以对传参长度有限制，不同浏览器上限不同（2k，7-8k或者其他），带重要信息的请求不合适，另外GET请求中，永远不会包含呈现数据

HEAD：和GET本质相同，区别在于HEAD不含有呈现数据，比如判断某个资源是否存在，这里HEAD的意义更明确

POST：和get不同，对传递资源的大小没有限制，取决于服务器端的接受能力，安全性稍高些，用途广泛，几乎目前所有的提交操作都是靠这个完成

PUT：比较少见，HTML表单不支持，和POST极为相似，都是像服务器发送数据，区别在于PUT指定了资源的存放位置，而POST没有，由服务器自己决定

DELETE：很少见，删除某一个资源，Amazon的S3云服务就用它来删除资源DELETE请求一般会返回3种状态码：
	200 (OK) - 删除成功，同时返回已经删除的资源
	202 (Accepted) - 删除请求已经接受，但没有被立即执行（资源也许已经被转移到了待删除区域）
	204 (No Content) - 删除请求已经被执行，但是没有返回资源（也许是请求删除不存在的资源造成的）

CONNECT：把服务器作为一个跳板，去访问其他网页，然后把数据返回回来，连接成功后，即可正常的GET，POST了

OPTIONS：极少使用，用于获取当前URL所支持的方法，请求成功会在HTTP头中包含“ALLOW”的头，值是所支持的方法

TRACE：极少使用，回显服务器收到的请求，主要用于测试和诊断，一般禁用，防止被恶意攻击或盗取信息




	                 状态码：
200 OK，当您的操作将在响应正文中返回数据时，出现此结果

204 No Content，当您的操作成功，但不在响应正文中返回数据时，出现此结果

304 Not Modified（重定向），当测试实体自上次检索以来是否被修改时，出现此结果

403 Forbidden，客户端错误

401 Unauthorized，客户端错误

413 Payload Too Large（客户端错误），当请求长度过长时，出现此结果

400 BadRequest（客户端错误），当参数无效时，出现此结果

404 Not Found（客户端错误），当资源不存在时，出现此结果

405 Method Not Allowed（客户端错误），由于方法和资源组合不正确而出现此错误。 例如，您不能对一个实体集合使用DELETE 或 PATCH

412 Precondition Failed，客户端错误

501 Not Implemented（服务器错误），当未实施某个请求的操作时，出现此结果

503 Service Unavailable（服务器错误），当 Web API 服务不可用时，出现此结果
